2) Conceptos clave (lo que te hace “pro”)


A) ¿Qué dispara un reflow? (layout recalculation)
Un reflow ocurre cuando algo puede cambiar tamaños o posiciones.

Disparadores comunes:

Cambiar width, height

padding, margin, border

Cambiar font-size (cambia el tamaño del texto)

Cambiar display

Cambiar position que afecta flujo

Cambios en layout de flex / grid

Cambiar el contenido (texto largo, imágenes cargando)

Cambiar tamaño de la ventana (resize)

📌 Regla mental:

Si afecta la geometría (tamaño/posición), hay reflow.



----------------------------


B) ¿Qué dispara un repaint? (solo repintar)

Repaint pasa cuando no cambia el layout, solo cambia cómo se ve.

Disparadores comunes:

background-color

color

border-color

outline-color

box-shadow (normalmente paint, puede ser pesado)

visibility (depende del caso, pero suele ser paint)

📌 Regla mental:
Si cambia el “look” pero no el tamaño/posición, es repaint.


-------------------

C) ¿Qué propiedades son “caras”?

“Caras” = consumen más CPU/GPU y pueden bajar FPS.

Muy caras (layout + paint)

Cambios frecuentes en:

width, height

top, left, right, bottom

margin, padding

display

✅ Si animas top/left → estás forzando layout constantemente.

Caras de paint

box-shadow grande

filter: blur(...)

backdrop-filter

gradients complejos

border-radius + sombras grandes en muchos elementos

Baratas (ideal para animar)

transform

opacity

-----------
📌 Regla pro:

Para animaciones suaves: usa transform y opacity.
